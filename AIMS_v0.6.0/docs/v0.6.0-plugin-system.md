# AIMS v0.6.0 - Plugin System

## Purpose

v0.6.0 adds a small but practical plugin architecture. The goal is to let AIMS run discipline-specific checks without hard-coding every future workflow into the core audit pipeline.

## New modules

```text
packages/aims_plugins/
  base.py
  registry.py
  loader.py

plugins/architecture/
  basic_arch_plugin.py

standards/plugins/
  plugin_manifest.yaml
```

## Plugin flow

```text
ADF Document
  -> Plugin Manifest YAML
  -> Plugin Loader
  -> Plugin Registry
  -> Plugin Execution
  -> Plugin Issues
  -> Markdown Report
```

## Built-in plugin

`architecture.basic` checks:

- open room boundaries
- room area below minimum
- door layer mismatch
- window layer mismatch

## CLI

```bash
aims audit examples/simple_site.dxf \
  --plugins-standard standards/plugins/plugin_manifest.yaml \
  --out reports/aims_report.md
```

## Plugin manifest example

```yaml
plugins:
  - name: architecture.basic
    path: plugins.architecture.basic_arch_plugin.BasicArchitecturePlugin
    enabled: true
    min_room_area: 1.0
```

## Design rule

Core AIMS should only understand ADF, validation, storage, and reporting. Discipline-specific behavior belongs in plugins whenever possible.
