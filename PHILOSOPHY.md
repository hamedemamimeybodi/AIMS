# AIMS Philosophy

Engineering data outlives software.

AIMS is built on the idea that engineering knowledge should remain portable, explainable, verifiable, repairable, and independent of the software that created it.

## Principles

- Domain before file format.
- Standards before implementation.
- Quality before feature count.
- Explainability before automation.
- Reversibility before auto-fix.
- Open specifications before closed workflows.
- Tests before trust.

## Project Signature

> We do not just validate drawings. We preserve engineering knowledge.
