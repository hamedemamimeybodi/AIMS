# AIMS v0.8.0 - IFC/OpenBIM Foundation

## Purpose

v0.8.0 introduces the first OpenBIM foundation layer for AIMS. The goal is not to claim full IFC authoring yet. That would be theatrical nonsense. The goal is to create a deterministic bridge from ADF/BIM-classified entities into an IFC-like STEP output, validate OpenBIM metadata, and prepare the codebase for a future IfcOpenShell-backed exporter.

## New modules

```text
packages/aims_openbim/
├── __init__.py
├── ifc_export.py
└── openbim_validation.py
```

## New standard

```text
standards/openbim/openbim.yaml
```

This file defines project identity and required OpenBIM property set values per IFC class.

## Pipeline update

```text
DXF
  ↓
ADF
  ↓
```

## CLI addition

```bash
aims audit examples/simple_site.dxf \
  --ifc reports/aims_output.ifc \
  --openbim-standard standards/openbim/openbim.yaml
```

## Scope of v0.8.0

- Validate missing IFC GUIDs.
- Validate spatial assignment through `level` or `spatial_container`.
- Validate required property-set values from YAML.
- Export IFC-like STEP records for:
  - IfcWall
  - IfcDoor
  - IfcWindow
  - IfcColumn
  - IfcSpace
  - IfcAnnotation
- Add OpenBIM readiness score to Markdown, HTML, and JSON reports.
