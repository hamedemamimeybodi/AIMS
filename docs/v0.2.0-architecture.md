# AIMS v0.2.0 Architecture Notes

## Goal

v0.2.0 upgrades AIMS from a basic DXF audit pipeline into an architectural QA/QC preflight package.

## Pipeline

```text
DXF
 ↓
DXF Parser
 ↓
ADF Document
 ↓
Classification
 ↓
Geometry + Architectural Validation
 ↓
SQLite Store
 ↓
Markdown Report
```

## Architectural QA Modules

- `metrics.py`: counts categories, layers, blocks, issue codes, and severity levels.
- `qa_score.py`: calculates a practical 0 to 100 quality score.
- `room_detection.py`: identifies closed room boundary candidates.

## Why ADF Stays Central

All future modules must depend on ADF, not directly on DXF. This keeps the system ready for DWG, IFC, DGN, LandXML, or custom imports later.
