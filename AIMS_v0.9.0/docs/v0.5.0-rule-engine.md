# AIMS v0.5.0 Rule Engine

AIMS v0.5.0 adds a YAML-driven rule engine. Validation logic can now live in editable standards files instead of being hard-coded into Python modules.

## New flow

```text
DXF -> ADF -> Built-in validators -> BIM/GIS engines -> Rule Engine -> SQLite/GeoJSON/Markdown
```

## Rule file

Default rule file:

```text
standards/rules/architectural_qaqc.yaml
```

A rule contains:

- `id`: stable issue code
- `selector`: which entities the rule applies to
- `severity`: info, warning, error, critical
- `message`: report text
- `conditions`: one or more checks that must pass

## Supported condition operators

- `exists`
- `not_empty`
- `equals`
- `in`
- `closed_geometry`
- `geometry_type`
- `min_area`
- `min_length`
- `block_name_matches`

## CLI

```bash
aims audit examples/simple_site.dxf \
  --rules-standard standards/rules/architectural_qaqc.yaml \
  --out reports/aims_report.md
```

## Purpose

This release prepares AIMS for project-specific standards, municipal rule packs, BIM preflight profiles, and future plugin-based validation.
